# 程序员面试知识库（Phase 1）

## 高频问题

1. 请介绍你最有代表性的项目，重点说明你负责的模块、技术选型和收益。
2. 当线上出现性能瓶颈时，你会如何定位并做权衡？
3. 如果要设计一个高并发系统，你会从哪些维度考虑？
4. 你如何保证代码质量和工程可维护性？

## 评分参考

- 专业能力：是否能结合场景说明设计理由与约束。
- 沟通表达：是否先给结论，再解释推理路径。
- 逻辑思维：是否有结构化拆解问题的能力。
- 问题解决：是否体现排障流程、回滚策略和复盘意识。

## STAR 情境追问

- 描述一次你推动跨团队协作并落地技术方案的经历。
- 描述一次你处理线上故障的完整流程（发现、分析、止损、复盘）。
