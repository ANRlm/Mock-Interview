# 程序员场景面试题库

## 复杂技术场景题

### 场景 1：分布式系统数据一致性

**背景**：你所在的公司计划将订单系统从单机数据库迁移到分布式数据库架构，需要在不停机的情况下完成数据迁移，并保证强一致性。

**挑战**：

- 如何设计数据分片策略，平衡负载与查询效率
- 如何处理跨节点事务，避免数据不一致
- 如何设计灰度迁移方案，确保故障可回滚

**请详细说明**：你会从哪些维度进行分析？技术选型的依据是什么？如何制定回滚预案？

### 场景 2：遗留系统技术改造

**背景**：你接手了一个已有 5 年历史的 Java 后端服务，代码耦合严重、文档缺失、测试覆盖率不足 30%。业务要求在 3 个月内上线新功能，同时不能影响现有服务稳定性。

**挑战**：

- 如何在有限时间内理清系统架构和核心逻辑
- 如何平衡新功能开发与技术债务偿还
- 如何建立有效的回归测试机制

**请详细说明**：你会采用什么策略分阶段推进改造？如何衡量改造进展？如何确保业务连续性？

### 场景 3：超大规模用户增长应对

**背景**：你负责的社区产品近期因营销活动，用户量在两周内从 50 万激增至 500 万，系统面临前所未有的压力。

**挑战**：

- 如何快速评估系统瓶颈和脆弱点
- 如何在有限时间内实施紧急扩容方案
- 如何设计常态化弹性架构以应对波动

**请详细说明**：你会优先处理哪些环节？如何与团队分工协作？长期来看需要哪些架构改进？

## 评分参考

- **问题拆解**：是否能将复杂问题分解为可处理的小问题
- **技术深度**：是否体现对分布式系统、架构设计的理解
- **权衡意识**：是否说明不同方案的取舍与 trade-off
- **实操经验**：是否结合真实经历或具体案例论证观点
